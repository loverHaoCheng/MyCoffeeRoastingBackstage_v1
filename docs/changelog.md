## 2026-07-15 - 备份错误提示与可选集合容错

- 主动备份遇到未初始化的可选历史集合时不再整体失败，缺失集合按 0 条写入备份摘要。
- 主动备份读取集合时若按 `created` 排序触发 PocketBase HTTP 400，会自动改为无排序读取；可选集合仍失败则按空集合处理。
- 上传备份遇到目标环境缺少可选集合时会跳过该集合，继续导入其他可用业务数据。
- 备份文件新增导出账号信息；同账号上传只按 PocketBase `id` 补充备份中多出的记录，跨账号上传会为所有备份记录重新分配 ID 后新增到当前账号。
- 同账号上传新增 `完全与备份同步` 与 `合并备份与账号数据` 两种方式；完全同步会更新同 ID 记录、补充缺失记录并删除备份外记录，适合正式端快照还原到测试端。
- 同账号上传的重复判断统一只参照 PocketBase `id`；跨账号存在相同编号或名称的生豆时仍保留为不同生豆，并将采购批次等关联数据挂到导入后映射出的对应生豆 ID。
- 统一所有业务集合的备份导入策略：同账号补充若查询不到旧 ID 但插入遇到 `id` 已存在或唯一冲突，会去掉旧 ID 后重新创建当前账号记录，避免 `app_settings` 单条冲突导致缺失设置被跳过。
- 收窄 `app_settings` 等可选集合的写入容错：读取失败仍可按空集合处理，但写入阶段不再把普通 HTTP 400 静默跳过，并统一移除导入记录里的 `created_at`、`updated_at` 时间元字段。
- PocketBase 返回 `Something went wrong while processing your request.` 时不再原样展示英文，统一转换为中文服务错误提示。
