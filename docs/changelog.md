# 变更记录

## 2026-07-11 - 发布流程并发保护

- `deploy.sh` 新增服务器范围的原子目录锁，阻止并发发布覆盖 BFF 临时/回退文件或前端入口。
- 发布锁保留 owner 信息，仅允许持有者释放；版本清理排除隐藏锁目录。
- 锁元数据增加创建时间、最近更新时间与执行阶段；遗留锁只允许受控人工恢复，不自动超时抢占。

## 2026-07-11 - 烘焙计划删除与通用计划选择修复

- 删除被历史烘焙记录引用的计划时，先解除记录中的计划 ID 关联并保留计划名称快照，再删除计划本身。
- 修复计划删除后被服务层本地镜像立即重新上传的问题；删除期间实时同步会跳过计划模块，成功后不再回填已删除计划。
- 烘焙计划后台同步改为 PocketBase 远端权威模式，不再自动上传远端已不存在的本地旧计划，确保后台或其他客户端删除能够持续生效。
- 通用烘焙计划无需预先选择生豆即可在烘焙创建、完整编辑和字段编辑中选择；实际保存烘焙记录前仍必须选择有库存的具体生豆。
- 统一数据卡片左侧标签固定预留 6 个汉字宽度，并与右侧值保持至少 `12px` 间距。

## 2026-07-11 - 认证数据清理与默认昵称

- 退出登录和账号注销后，前端会清理当前站点的 Web Storage、可访问 Cookie、IndexedDB 与 Cache Storage；认证 `HttpOnly Cookie` 继续由 BFF 响应过期。
- 新注册用户仅在首次创建 PocketBase 用户时，以注册邮箱初始化 `users.name`；用户后续可自行修改昵称，系统不会再同步覆盖。
- 统一数据卡片的左侧字段标签固定为单行，空间不足时省略显示，保证右侧多行值不会带动标签换行。

## 2026-07-11 - 开发代理切换到 HTTPS 域名

- 本地 Vite `/api/*` 开发代理默认目标改为 `https://www.easybake.top`
- 更新 `.env.example`、本地 `.env.local` 与登录态文档，避免开发环境继续代理到裸 IP 导致 `/api/auth` 返回 HTML 错误页
- Auth BFF 默认 PocketBase 地址同步改为 `https://www.easybake.top`，减少服务器环境变量遗漏时的 404 风险

## 2026-07-11 - 正式发布脚本验证

- `deploy.sh` 上传完成后会验证正式站点 `https://www.easybake.top` 的 `version.json` 与 `/api/health`
- 发布脚本支持通过 `APP_URL` 和 `REMOTE_TARGET` 覆盖正式访问地址与 rsync 目标，默认用户访问走 HTTPS 443

## 2026-07-11 - PocketBase 同源默认地址

- 前端 PocketBase 默认地址改为当前浏览器同源地址，避免生产构建未配置 `VITE_PB_URL` 时回落到 HTTP 裸 IP
- 更新 PocketBase 部署说明，明确只有前端与 PocketBase 分域部署时才需要设置 `VITE_PB_URL`

## 2026-07-10 - 设置页账号注销开放

- 设置页底部 `注销账号` 危险按钮默认开放，点击后进入原有 5 秒倒计时确认流程
- 更新设置页账号注销交互文档，移除发布前禁用说明

## 2026-07-10 - 生豆 AI 图片识别服务端基础

- 新增 `/api/ai/bean-image-recognition` BFF 接口，服务端代调用七牛云 OpenAI 兼容 Qwen 多模态接口
- 新增按用户月度额度控制的服务端流程，读取 `ai_usage_limits`，默认 `10 次/月`
- 新增 AI 使用日志写入规则，只有 `success` 计入额度，失败写 `failed` 但不扣次数
- 账号注销时同步清理 `ai_usage_limits` 与 `ai_usage_logs`
- 生豆新增抽屉的 AI 图片识别 Tab 支持上传图片、预览识别字段，并一键回填到手动创建表单
- 更新 API、PocketBase 集合说明与生豆新增交互文档

## 2026-07-09 - 财务总览落地

- 新增 `#/finance` 一级页面，提供经营总览、手工收入台账、经营费用台账与成本核算记录入口
- 新增 `useFinanceOverview` 聚合口径，统一输出全部花费、已实现收入、当前库存预估收入、毛利润、经营利润及对应利润率
- 新增收入台账与经营费用台账表单，并统一切换到 PocketBase 财务台账集合
- 成本核算也收紧为 PocketBase 主链路，不再保留本地兜底记录
- `#/production` 顶部新增“本月经营快照”，可直接跳转财务页
- 更新 README、架构说明、数据模型、接口约定与交互文档，补充财务模块当前口径和显示位置

## 2026-07-09 - 财务链路清理

- 删除已无入口的手工收入台账前端链路，已实现收入统一只从烘焙历史销售记录聚合
- 抽出财务支出类别与标题的共用格式化工具，减少页面、服务和表单中的重复逻辑
- 清理测试环境下的预期日志噪音，并修复 `RoastPage` 测试中的 `act(...)` 包裹缺失

## 2026-07-08 - Supabase 清理收口

- 移除生豆主库与成本核算对 Supabase 的依赖，主业务数据统一回到 PocketBase 主库
- 保留熟豆 Supabase 的两条链路：设置页连接配置与烘焙历史创建后的熟豆镜像写入
- 删除历史 Supabase 初始化脚本与迁移脚本，更新相关架构、数据模型与交互文档
- 修复烘焙历史与烘焙计划在乐观创建期间被后台同步重复上传的问题；`syncLocalAndRemote()` 现在会跳过 `local-*` 临时记录，并在刷新后继续保留这些乐观记录直到真实创建完成
- 设置页顶部账号栏支持昵称编辑：顶部主文案改为昵称，邮箱降级为辅助信息，并可通过小型字段编辑抽屉把昵称保存到 PocketBase `users.name`
- 生豆主档新增“风味”多标签字段：前端以 chips 方式录入与展示，主库以 `flavor_tags` 逗号字符串存储，搜索支持模糊包含匹配，并会在创建烘焙历史时同步到熟豆 Supabase 的 `flavor: string[]`
- 生豆主档新增“烘焙后处理”分组，包含 `养豆时间` 与 `赏味结束期` 两个天数字段，默认分别为 `14` 与 `40`，支持列表卡片展示与单字段编辑，并在创建烘焙历史时同步到熟豆 Supabase 的 `startDay/endDay`
