# EasyBake 数据删除机制

> 生效日期：2026-07-09  
> 运营主体：个人开发者  
> 联系邮箱：zhc1501705072@163.com  
> 适用范围：账号注销、个人信息删除、业务数据删除、备份数据处理  
> 说明：本文档是产品和工程执行机制，正式上线前建议结合真实部署环境复核。

## 1. 可删除范围

用户可申请删除的数据包括：

- 账号邮箱、账号资料、邮箱验证状态等账号信息。
- 生豆、供应商、客户、烘焙计划、烘焙批次、库存、生产、财务、成本模板等业务数据。
- 用户设置、界面偏好、同步配置、待同步缓存等辅助数据。
- 与账号关联且不再有必要保存的认证记录和运行记录。

因法律法规、安全审计、争议处理或备份恢复需要暂时留存的数据，应按最小必要原则限制访问，并在留存目的消失后删除或匿名化。

## 2. 申请方式

用户可通过设置页 `账号与数据 -> 注销账号` 发起自助注销。

用户也可使用注册邮箱发送申请至 `zhc1501705072@163.com`。

邮件标题建议：

- `EasyBake 数据删除申请`
- `EasyBake 账号注销申请`

邮件正文建议包含：

- 注册邮箱。
- 希望删除的数据范围。
- 是否需要注销账号。
- 是否需要在删除前导出数据。

## 3. 身份核验

运营者优先通过来信邮箱是否等于注册邮箱进行核验。

如申请内容涉及高风险操作或账号归属不明确，可要求用户补充可证明账号归属的信息，但不得要求提交与核验无关的敏感信息。

## 4. 处理时限

自助注销确认后，系统会立即执行删除流程。

收到邮件申请并完成身份核验后，原则上在 15 个工作日内完成处理并通过邮件反馈。

如因数据量、备份周期、系统维护或法律原因无法按期完成，应告知原因和预计完成时间。

## 5. 删除执行原则

账号注销时，应删除与该账号关联的业务数据、设置数据、待同步缓存和认证记录。

删除前应确认是否需要用户自行导出或备份数据；删除完成后，相关数据通常无法恢复。

后端实现应以用户 ID 为边界执行级联删除或匿名化，不得跨用户影响其他账号数据。

## 6. PocketBase 落地建议

当前项目使用 PocketBase Auth 和业务集合时，后续应补齐服务端删除接口或管理脚本：

1. 根据注册邮箱查询 `users` 记录，确认用户 ID。
2. 按用户 ID 查询并删除或匿名化该用户在各业务集合中的记录。
3. 删除用户设置、同步配置、待同步缓存等辅助集合记录。
4. 使该用户所有会话失效。
5. 最后删除或禁用 `users` 账号记录。

如果某些集合存在被财务、库存或审计链路引用的数据，应优先选择匿名化或软删除，并在数据模型文档中标注原因。

## 7. 处理记录

运营者应记录：

- 申请时间。
- 申请邮箱。
- 处理范围。
- 核验结果。
- 完成时间。
- 处理人。
- 异常原因或延期原因。

处理记录仅用于安全、合规和争议解决，不用于营销或与删除请求无关的用途。

## 8. 前端入口要求

当前已提供公开页面：

- `#/terms`
- `#/privacy`
- `#/data-deletion`

注册页必须要求用户勾选已阅读并同意《用户协议》和《隐私政策》后才能提交注册。

当前设置页已提供“账号与数据”危险操作区，支持 5 秒倒计时后执行不可恢复的自助注销。
